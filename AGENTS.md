# コーディングスタイル

- 関数は、単一責任の原則に従い、1つのことだけを行うように設計すること。
- 関数には*必ず*TSDocコメントをつけること。
- 保守しやすいように、*頻繁に*コメントを入れること。
- 仕様に無いフォールバックは行わないこと。
- 3項演算子は1行に収め、ネストさせないこと。
- 変数名の英単語は、`str` や `i` などのよく使われるものを除き、省略しないこと。
- `if` 文の中括弧は省略しないこと。
  ```ts
  // NG
  if (condition) doSomething();
  // OK
  if (condition) {
    doSomething();
  }
  ```
